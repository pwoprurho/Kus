Core Ecosystem Workflows

A

In-App Hiring

1
Discovery

User browses the Marketplace, which reads registry.json from the local kushub-mirror.

2
Hiring

Kusecurity signs the request and performs a git sparse-checkout of the skill folder.

3
Nano-Activation

nanoclaw hot-reloads the skill, minifying to ≤300 tokens. The specialist is now a "Sovereign Pal."

B

Sovereign Announcement Protocol

1
Forge Update

A new insight or update is pushed to announcements.json in the kushub repo.

2
Field Sync

The app fetches the signed JSON. Kusecurity verifies the Ed25519 signature using the Forge public key.

3
Broadcast

Validated updates appear in the Mission Control tab with priority-coded badges.

C

Recursive Learning

1
Trace Capture

Autodidatic records the "Thought Steps" of every specialist task into structured JSON traces.

2
PRM Gating

The Process Reward Model scores each step for accuracy. Only traces scoring ≥0.7 pass.

3
MadMax DPO

At 3:00 AM, the scheduler fine-tunes the local 1.5B model using LoRA adapters and high-score traces.

Key Architectural Decisions

01

Real Cryptography

Kusecurity Passport uses ed25519-dalek (Rust) and Python's cryptography library — not placeholder crypto. Every SAP message is verifiable.

02

Token Budget Enforced

nanoclaw's skill minifier actively truncates SKILL.md content to ≤300 tokens with measured compression metrics. No skill exceeds the budget.

03

KV-Cache Intelligence

The proxy tracks per-context memory impact (2 bytes × tokens × 32 layers) and rejects requests that would breach the 900MB ceiling.

04

Weighted PRM Scoring

Later reasoning steps carry exponentially more weight in the aggregate score, matching how real logical reasoning builds on earlier steps.

05

Mobile-Optimized DPO

LoRA rank 8 + FP16 + gradient checkpointing keeps the training footprint under ~2GB during Quiet Hours on 8GB devices.
